# Silver Memory Monetization

Silver Memory의 수익화는 추모의 진정성을 해치지 않는 순서로 붙인다. 첫 화면의 목적은 결제가 아니라 기억을 보는 것이고, 유료 기능은 가족과 방문자가 원할 때만 조용히 드러나야 한다.

## 1. 방문자 1회성 후원

- 작은 꽃: 4,000원대
- 추모 꽃다발: 20,000원대
- 해외 조문객을 위해 USD 가격대도 함께 제공한다.
- 공개 목록에는 금액을 노출하지 않는다.
- 유족에게 전달되는 금액, 수수료, 서비스 운영비를 결제 전에 분리해서 보여준다.
- 결제 전 화면에서 환불 가능 조건, 환불 불가 조건, 처리 예상 시간을 명확히 보여준다.
- 결제 성공 직후 일정 시간 안의 실수 결제는 유족 정산 전 자동 환불 대상으로 둔다.
- 유족이 메시지를 숨기더라도 결제/환불 상태는 방문자와 관리자에게 분리해서 설명한다.
- 결제 전 프로토타입에서는 "시범으로 남기기"만 제공한다.
- 초반 명칭은 법적/정산 부담이 큰 "조의금 송금"보다 "디지털 헌화"로 둔다.

## 2. 유족 프리미엄 추모관

- 월 구독 또는 연 단위 보관료
- 커스텀 도메인, 고급 전시관 템플릿, 대용량 사진/영상, 백업 내보내기
- 가족 편집 권한, 편집 기록, 공개 범위 관리 고도화
- AI 태그, 생애 요약, 사진 설명 자동 생성은 선택형 부가 기능으로 둔다.

## 3. 장례식장/B2B 패키지

- 장례식장, 봉안당, 추모공원, 반려동물 장례 업체 대상
- QR 카드 제작, 유족 초기 세팅, 안내문 템플릿, 관리자 대시보드 제공
- 월 사용료 + 추모관 생성 건당 과금 모델을 검토한다.
- 제휴 랜딩은 `#/business`에서 제공한다.
- 초기 파일럿은 "샘플 추모관 1개 + QR 카드 이미지 + 유족 안내 문구 + 직원 설명 스크립트"로 작게 판매한다.
- 매출 항목은 초기 세팅비, 추모관 생성 건당 비용, 월 파트너 요금, QR 카드/스티커 제작 마진으로 분리한다.
- 방문자 결제는 바로 붙이지 않고, B2B 파일럿에서는 QR 추모관 생성과 유족 편집 경험을 먼저 검증한다.
- 제휴 문의는 `POST /api/memory/partner-inquiries`로 저장한다.
- 공개 페이지에서는 문의 목록을 노출하지 않는다. 관리자 조회는 인증/권한 설계를 먼저 붙인 뒤 별도 화면으로 연다.

## MVP 순서

1. "남겨진 꽃" 시범 UI로 반응을 확인한다.
2. `memory_tribute_orders` 테이블과 결제/환불 상태 모델을 추가한다.
3. `POST /api/memory/memorials/{slug}/tribute-orders`로 결제 준비 주문을 만든다.
4. 결제 provider는 한 곳만 먼저 붙인다.
5. 결제 성공 후 꽃을 공개하고, 실패/환불/취소 상태는 방문자/유족/관리자 화면에서 구분해 확인한다.
6. 유족 프리미엄과 B2B 패키지는 꽃 놓기 지표를 본 뒤 상품화한다.
7. 장례식장 제휴 문의가 생기면 `#/business` 랜딩의 문구와 패키지명을 실제 영업 반응에 맞춰 조정한다.

## 결제/환불 상태 모델

`memory_tribute_orders`는 처음부터 결제와 환불을 함께 저장한다. 공개 페이지에는 `paid`, `partially_refunded` 상태의 꽃만 노출하고, `refunded`, `canceled`, `failed` 상태는 공개 목록에서 숨긴다.

금액은 통화별 최소 단위로 저장한다. `KRW 4000`은 4,000원, `USD 400`은 4달러를 의미한다.

- `draft`: 결제창 진입 전 임시 주문
- `pending`: PG 결제 요청 생성
- `paid`: 결제 성공, 공개 가능
- `failed`: 결제 실패
- `canceled`: 결제 전 취소 또는 승인 전 취소
- `refund_requested`: 환불 요청 접수
- `partially_refunded`: 부분 환불 완료, 공개 여부는 관리자 정책에 따른다
- `refunded`: 전액 환불 완료, 공개 목록에서 숨김
- `refund_rejected`: 환불 거절, 사유를 관리자/요청자에게 남김

환불 요청 사유는 최소한 다음을 지원한다.

- 결제 실수 또는 중복 결제
- 메시지 오입력
- 유족 요청
- 부적절한 메시지 또는 운영정책 위반
- PG 오류 또는 정산 전 취소

운영 원칙:

- 유족 정산 전 주문은 가능한 한 빠르게 환불한다.
- 정산 이후 환불은 플랫폼 부담, 유족 차감, 부분 환불 중 하나로 처리 방식을 명확히 선택해야 한다.
- 환불되어도 회계/감사 목적의 주문 레코드는 삭제하지 않고 상태만 바꾼다.
- PG webhook은 멱등 처리하고 `provider_payment_id`, `provider_order_id`는 중복 저장을 막는다.
- 환불 정책과 개인정보 처리 안내를 결제 버튼 바로 위에 노출한다.

## 글로벌 헌화 확장

- 1차 지원 통화는 `KRW`, `USD`로 제한한다.
- 공개 카드에는 금액을 표시하지 않거나, 유족이 켠 경우에만 통화 없는 상품명만 보여준다.
- 국가별 세금, 환율, 송금 규정이 정리되기 전까지는 유족에게 바로 송금되는 "조의금"보다 플랫폼이 제공하는 디지털 상품으로 운영한다.
- 해외 카드 결제는 PG 정산 통화, 환율, 차지백 가능성을 고려해 국내 결제와 별도 모니터링한다.
- 향후 유족 정산형 조의금으로 확장할 때는 유족 KYC, 정산 계좌 검증, 환불/차지백 준비금을 별도 모델로 둔다.
